Accessibility Statement for TechBlog

This is an accessibility statement from WebAble Crew - Group 10.

Measures to support accessibility

WebAble Crew - Group 10 takes the following measures to ensure accessibility of TechBlog:

  • Assign clear accessibility goals and responsibilities.
  • Employ formal accessibility quality assurance methods.
  • We integrate accessibility into development by using semantic HTML, ARIA landmarks, keyboard navigation, skip links and responsive design to enhance assistive technology compatibility.

Conformance status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA. TechBlog is partially conformant with WCAG 2.1 level AA. Partially conformant means that some parts of the content do not fully conform to the accessibility standard.

Compatibility with browsers and assistive technology

TechBlog is designed to be compatible with the following assistive technologies:

  • We recommend using recent versions of Chrome, Firefox, or Edge with updated assistive technologies (e.g., NVDA 2024, VoiceOver on macOS Ventura, or TalkBack on Android 12+) for optimal compatibility.

TechBlog is not compatible with:

  • Internet Explorer 11 may not render the website correctly because of modern CSS (Tailwind CSS) and JavaScript (ES6+).
  • JAWS 2018 may struggle with dynamic content updates due to outdated ARIA support.

Technical specifications

Accessibility of TechBlog relies on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your computer:

  • HTML
  • WAI-ARIA
  • CSS
  • JavaScript
  • SVG

These technologies are relied upon for conformance with the accessibility standards used.

Limitations and alternatives

Despite our best efforts to ensure accessibility of TechBlog , there may be some limitations. Below is a description of known limitations, and potential solutions. Please contact us if you observe an issue not listed below.

Known limitations for TechBlog:

  1. Text contrast on post content: Text in red color may be not readable for users with impairments, color blindness, dyslexia or elderly because the issue is caused by a CDN-hosted stylesheet used for syntax highlighting. This stylesheet applies low-contrast colors that do not meet WCAG 2.1 AA for normal text (4.5:1 ratio). We are investigating alternative syntax highlighting themes with higher contrast ratios compatible with atom-one-dark.min.css. Please enable high-contrast mode in browsers or use screen readers to access content auditorily.
